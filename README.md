# 🔌 מערכת ניהול השבתות — Shutdown Manager

מערכת פנים-ארגונית לניהול השבתות מתוכננות: תיאום מועד, קבלת אישורים מהגורמים המושפעים,
דיון בצ'אט בזמן אמת, לוח שנה עברי-לועזי, וסיכום עם לקחים. בנויה לרוץ **ברשת סגורה** ללא תלות באינטרנט.

> **הערה על מצב נוכחי:** המערכת רצה כרגע מקומית (לא ברשת סגורה). כל החיבורים החיצוניים
> (מייל/Outlook, שליפת משתמשים מ-AD) בנויים כ**מודולים רדומים** שמופעלים בפעולות פשוטות בזמן
> המעבר לרשת הסגורה — ראו הסעיף [🔌 חיבור לרשת סגורה](#-חיבור-לרשת-סגורה--נקודות-אינטגרציה).

## סטאק
- **שרת**: Node.js + Express + Socket.IO + `node:sqlite` (SQLite מובנה — אפס תלויות native)
- **קליינט**: React + Vite (RTL, עברית), PWA
- **אבטחה**: JWT ב-httpOnly cookie, bcrypt, helmet, rate-limit, zod, ניתוק מיידי באיפוס סיסמא
- אפס תלות ברשת בזמן ריצה — פונטים ונכסים מקומיים, נבדק אוטומטית באריזה

## הרצה מהירה (פיתוח)
```bash
npm install
npm run dev          # שרת (3000) + קליינט (5173) במקביל
```
פתחו http://localhost:5173 . לבנייה לפרודקשן: `npm run build` ואז `npm start` (השרת מגיש את הקליינט הבנוי מפורט 3000).

## פיצ'רים
- **קבוצות והרשאות** — 3 רמות: מנהל מערכת (admin), מנהל השבתה (פר-קבוצה), משתמש
- **השבתות** — יצירה, תגובה (✓ אישור / ✗ דחייה / ~ מותנה) עם תאריך חלופי ומשמעות אישית, קיבוע תאריך סופי, מעברי סטטוס אוטומטיים לפי שעה
- **צ'אט בזמן אמת** פר-השבתה — היסטוריה, אזכורים @, חיפוש, הודעות מנהלה, הודעות פרטיות, סנכרון אחרי ניתוק
- **התראות** — קופצות בזמן אמת, פעמון, תזכורות יומיות, באנר "השבתה בביצוע"
- **לוח שנה עברי-לועזי** + ייצוא ICS ל-Outlook
- **קבצים** — צירוף קבצים להשבתה + עמוד "נהלי השבתות" (מצגות/מדריכים)
- **צ'קליסט** (כולל פרטי למנהלים), **משוב מכל המשתתפים**, **מסמך מרוכז** מקצועי להדפסה/PDF
- **דוחות** — ייצוא CSV, גרף חודשי, יומן פעולות (audit)
- **גיבוי לילי אוטומטי** (DB + קבצים)

## משתמשי ברירת מחדל
נוצרים אוטומטית בהרצה ראשונה — **חובה להחליף סיסמאות**:

| משתמש | סיסמא | תפקיד |
|---|---|---|
| `admin` | `admin123` | מנהל מערכת |
| `Linor` | `Linor123` | מנהלת מערכת (הרשאות מלאות) |

## אריזה ופריסה לרשת סגורה
```bash
npm run package      # יוצר dist-deploy/ עצמאי לחלוטין
```
מעתיקים את `dist-deploy/` לשרת היעד (נדרש רק Node.js 22.13+), ומריצים `start.bat` או מתקינים
כשירות Windows עם `install-service.bat` (NSSM). הקונפיגורציה ב-`server/.env` (ראו `server/.env.example`).
האריזה כוללת בדיקת **אפס-קריאות-חיצוניות** אוטומטית.

## 🔌 חיבור לרשת סגורה — נקודות אינטגרציה

כל חיבור חיצוני בנוי כמודול **config-gated** שרדום עד שממלאים הגדרות. לאיתור כל הנקודות בקוד:
```bash
grep -rn "INTEGRATION: closed-network" server/
```
מדריך מפורט: [`docs/CLOSED-NETWORK.md`](docs/CLOSED-NETWORK.md). טאב **"חיבורים / רשת"** בעמוד הניהול מציג סטטוס חי.

| # | אינטגרציה | סטטוס | מה צריך לעשות | קבצים |
|---|---|---|---|---|
| 1 | **מייל (Exchange/SMTP)** | ⬜ נדרשת פעולה | למלא `SMTP_HOST/PORT/FROM` ב-`.env` | `server/src/services/mailer.js` |
| 2 | **שליפת משתמשים (Active Directory)** | ⬜ נדרשת פעולה | `npm i ldapts -w server`, למלא `LDAP_*`, ולממש את `authenticateViaDirectory()` | `server/src/services/directory.js`, `server/src/routes/auth.js` |
| 3 | **ייצוא ל-Outlook (יומן)** | ✅ מוכן | ללא פעולה — מנוי `webcal` ל-`/api/calendar/my.ics` | `server/src/routes/calendar.js` |

**עקרונות בטיחות:** `admin` ו-`Linor` נשארים משתמשים מקומיים תמיד — גם אם ה-Domain Controller לא זמין, לא ניתן להינעל מחוץ למערכת. כל אינטגרציה שכבויה = המערכת פשוט עובדת בלעדיה, ללא שגיאות.
