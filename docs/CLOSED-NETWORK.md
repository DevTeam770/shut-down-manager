מסמך זה מרכז את כל נקודות האינטגרציה של המערכת עם סביבה ארגונית סגורה, ומה בדיוק נדרש
כדי להפעיל כל אחת. **כרגע כולן רדומות** — המערכת עובדת מקומית בלי אף אחת מהן.

איתור כל הנקודות בקוד:
```bash
grep -rn "INTEGRATION: closed-network" server/
```
כל נקודה מסומנת בקוד בהערה `// [INTEGRATION: closed-network]` עם הסבר צמוד.

## 1. מייל דרך Exchange / SMTP פנימי

**מטרה:** התראות והודעות מנהלה מגיעות גם למייל Outlook של המשתמשים.
נשלח מייל על: השבתה חדשה · שינוי תאריך · קיבוע תאריך סופי · תזכורת יומית · הודעת מנהלה
לכולם · הודעת מנהל בחדר הצ'אט · מסמך מרוכז כשכולם מאשרים.

**קובץ:** `server/src/services/mailer.js` (המנוע), נקרא מ-`services/events.js` ומ-`sockets/chat.js`.

**הפעלה (פעולה פשוטה):** מלאו ב-`server/.env`:
```
SMTP_HOST=mail.corp.local
SMTP_PORT=25
SMTP_FROM=shutdown-manager@corp.local
# SMTP_USER=   (אם ה-relay דורש אימות)
# SMTP_PASS=
```
בנוסף — לכל משתמש שאמור לקבל מייל יש למלא כתובת מייל (עמוד ניהול → עריכת משתמש → שדה מייל,
או אוטומטית דרך AD בסעיף 2). ריסטארט לשירות. זהו.

**איך לוודא:** טאב "חיבורים / רשת" בעמוד הניהול יראה ✅ ל"מייל".

## 2. שליפת/אימות משתמשים מ-Active Directory (LDAP)

**מטרה:** עובדים נכנסים עם **שם המשתמש והסיסמא של הרשת הארגונית**, בלי הרשמה ידנית.
בכניסה ראשונה נוצר אוטומטית משתמש מקומי (auto-provision) עם השם והמייל מה-AD.

**קבצים:** `server/src/services/directory.js` (המתאם) · נקודת החיבור ב-`server/src/routes/auth.js` (login).

**מצב נוכחי:** המתאם קיים ומחווט ל-login, אך גוף האימות מחזיר `null` (רדום). ההתחברות המקומית עובדת כרגיל.

**הפעלה (3 צעדים):**
1. התקנת הספרייה (מהמכונה עם גישה, לפני האריזה לרשת הסגורה):
   ```bash
   npm i ldapts -w server
   ```
2. מילוי `server/.env`:
   ```
   LDAP_URL=ldap://dc1.corp.local:389        # או ldaps://...:636 (מוצפן, מומלץ)
   LDAP_BASE_DN=DC=corp,DC=local             # חילוץ: whoami /fqdn על מחשב בדומיין
   LDAP_UPN_SUFFIX=@corp.local               # מה שאחרי ה-@ בכניסה לווינדוס
   # LDAP_REQUIRED_GROUP=CN=ShutdownUsers,... # אופציונלי: רק חברי קבוצה זו יורשו
   ```
3. מימוש גוף הפונקציה `authenticateViaDirectory()` ב-`directory.js` — הקוד המלא כבר כתוב שם
   כהערה מוכנה-להעתקה תחת `// [INTEGRATION: closed-network] TODO` (bind + search של displayName/mail/memberOf).

**מה קורה בכניסה:** אם `LDAP_URL` מוגדר → ניסיון אימות מול ה-DC → provision + הנפקת JWT.
כשל/כבוי → נפילה חלקה להתחברות מקומית. **admin ו-Linor תמיד מקומיים** — אין נעילה בחוץ.

**הערה:** את החיבור האמיתי אפשר לבדוק רק מול Domain Controller אמיתי (בסביבתכם), לא בפיתוח.

## 3. ייצוא / סנכרון ל-Outlook (יומן)

**מטרה:** ההשבתות מופיעות ביומן Outlook של המשתמש.

**קובץ:** `server/src/routes/calendar.js`. **סטטוס: ✅ מוכן — אין צורך בהגדרה.**

**שימוש:**
- **השבתה בודדת:** כפתור "📅 הוספה ל-Outlook" בדף ההשבתה מוריד קובץ `.ics`.
- **מנוי לכל ההשבתות שלי:** ב-Outlook → File → Account Settings → Internet Calendars → New →
  להדביק את הכתובת `http://<שרת>/api/calendar/my.ics` (או `webcal://...`). היומן יתעדכן אוטומטית.
